GitHub Project Setup Guide:

Follow these initial steps to set up your team for project collaboration on GitHub.

  1. Initial Repository Setup

  • Create Your Private Repository - Start with a dedicated repository for all project code and planning.

  • Invite Your Team & Me

  • Go to Settings -> Collaborators -> Add People

    • Add all teammates with “Write” access

    • Add me too: mailmanoov@gmail.com with “Write” access

    • This allows me to review code, monitor progress, and help in real-time.

  1. Task Management with GitHub Issues Treat Issues as your central hub for ideation, research, and task definition.
  • Break your project into small, clear tasks
  • Detailed Description: Each issue must clearly state the goal and required outcome, along with the Assignee (who‚Äôs responsible).
    • Sub-Issues & Task Lists: Use the Sub-Issue feature for dependencies and Markdown checklists (- ) within descriptions to track granular progress.
  • Collaboration: Discuss details, and decisions proactively in the comments section.
  1. Track Progress Visually with GitHub Projects

    • Project Integration: Link all your Issues (including sub-issues) to a GitHub Project. GitHub Projects

    • Visualization: Use the Project’s customizable board (Kanban or Table view) to track the flow of work (To Do, In Progress, Review, Done).

  2. Branching & PRs To ensure your contributions are tracked and graded, follow this workflow for every task.

Step 1: Update your local code/README.md Before starting, ensure you have the latest version of the project from the main branch.

git checkout main
git pull origin main

Step 2: Create a new branch Never work directly on main. Create a branch for your specific task.

# Replace 'task-name' with your actual task
git checkout -b student-name/task-name

Step 3: Save your work (Commit) As you work, save your progress with clear, descriptive messages. Small, frequent updates prevent “merge chaos.”

git add .
git commit -m "Brief description of what you changed"

Step 4: Upload your branch to GitHub

git push origin student-name/task-name

Step 5: Create a Pull Request (PR) - Go to your repository on GitHub.com.

- Click the yellow bar saying "Compare & pull request."

- Link the Issue: In the description, write Closes #IssueNumber (e.g., Closes #4). This automatically links your code to the task.

- Click "Create pull request."

- Commit and push regularly - don't save everything for last minute

  1. Grading checklist and Rubric
Category Weight Criteria
Commit Consistency 25% Shows steady, incremental work throughout the project timeline (no “last-minute” bulk uploads).
Technical Contribution 30% Code or documentation submitted via Pull Requests that meet the “Files Changed” standards.
PR Communication 20% Clear, descriptive Pull Request messages that link to Issues (e.g., Closes #1) and explain the “Why”.
Peer Review 15% Actively engaged in commenting on and reviewing at least one teammate’s code or documentation.
Project Management 10% Properly uses the GitHub Project board and Issues to track tasks, deadlines, and progress updates.
